## 配置mingw压缩包

下载地址: https://github.com/niXman/mingw-builds-binaries/releases
下载版本: x86_64-15.2.0-release-win32-seh-ucrt-rt_v13-rev0.7z

解压配置环境变量(可选), 也可以不配置环境变量, 下文我全都是给vscode配置全路径识别的
```
MINGW_HOME: C:\Users\hasee\.mingw\x86_64-15.2.0-release-win32-seh-ucrt-rt_v13-rev0\mingw64
PATH: %MINGW_HOME%\bin
```

验证环境变量
```
gcc -v
gcc --verison
g++ --version
g++ -v
```

## 编译运行

编译测试文件
```
##编译单个C++文件：g++ -g "C++源文件" -o 编译生成的可执行文件，例如：
g++ -g .\main.cpp -o main.exe。
##如果不通过-o指定输出的exe名称, 则默认生成a.exe 
g++ main.cpp
```
编译好的产物a.exe直接执行
```
.\a.exe
```

当main.cpp通过头文件引用了三方的cpp实现的时候, 如main.cpp引用了头文件a.h并调用了头文件中的函数func, func的实现是func.cpp

**你需要同时编译多个cpp文件(main.cpp和func.cpp), 否则会提示你资源缺失**
```
##编译多个C++文件：g++ -g "C++源文件1" "C++源文件2" ... "C++源文件n" -o 编译生成的可执行文件，例如：
g++ -g a.cpp b.cpp c.cpp -o multi_file.exe。
```

上面命令可执行文件后缀名(.exe)可以省略。
