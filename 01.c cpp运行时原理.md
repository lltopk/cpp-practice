C/C++ 程序要变成可运行的文件（windows下为 .exe） ，要经历 四个阶段：预处理 → 编译 → 汇编 → 链接。

（部分工具链会自动合并执行某些阶段）

以一个简单的 hello.cpp 为例：
```
#include <iostream>
int main() {
    std::cout << "Hello, World!" << std::endl;
    return 0;
}
```
想要把hello.cpp跑起来, 需要经历以下阶段
```
hello.cpp
    │
    ▼
[预处理器] → hello.ii
    │
    ▼
[编译器]   → hello.s
    │
    ▼
[汇编器]   → hello.obj
    │
    ▼
[链接器]   → hello.exe  ← 链接 CRT、C++ 标准库、Windows SDK 等
```

## 预处理（Preprocessing）

预处理工作如下：

处理所有以 # 开头的指令。
- 展开 #include（如 <iostream>）
- 替换 #define 宏
- 根据 #ifdef 等条件编译指令决定保留哪些代码

输出：一个“纯”C++ 文件（无宏、无头文件引用），通常以 .i（C）或 .ii（C++）为扩展名。

## 编译（Compilation）
编译工作如下：

将预处理后的 C/C++ 代码 翻译成汇编语言（Assembly）。

输出：平台相关的汇编代码文件（.s 或 .asm）。

特点：此阶段进行语法检查、类型检查、优化等。

## 汇编（Assembly）

汇编工作如下：

将汇编代码 转换为机器码（二进制目标代码）。

输出：目标文件（Object File），扩展名为 .o（Linux/macOS）或 .obj（Windows）。

内容：包含函数、变量的二进制代码，但尚未解决外部引用（如 printf、std::cout）。

## 链接（Linking）

链接工作如下：

将一个或多个 .obj 文件与系统库（如 CRT、C++ 标准库、Windows API）合并，生成最终的可执行文件。

具体是解析未定义符号（如 main 调用了 std::cout，需从标准库中找到其实现）, 合并代码段、数据段

生成 Windows PE（Portable Executable）格式的 .exe 文件

输出：hello.exe
